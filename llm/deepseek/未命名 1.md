# `tests/unit_tests/modules/test_cp.py` 单元测试总结

## 背景与 CP 分支对应关系

| 模型 | CP 分支 | 实现模块（本文件引用） |
|------|---------|------------------------|
| DeepSeek V32 | **DSA** 分支 | `torchtitan_npu.distributed.context_parallel.dsa_cp`；注意力侧为 `DSV32_SDPA`（`torchtitan_npu.models.deepseek_v32.model.model`） |
| DeepSeek V3 | **Ulysses** 分支 | `torchtitan_npu.distributed.context_parallel.ulysses_cp`；注意力侧为 `torchtitan.models.attention.ScaledDotProductAttentionWrapper` |

本文件在单测中分别验证：**DSA CP** 下的序列 all-gather、DTensor 梯度、`DSV32_SDPA` 的 patch 与 `dsa_forward_with_cp` 入参校验；**Ulysses CP** 下的 `all_to_all`、`ScaledDotProductAttentionWrapper` 的 patch 与 Ulysses 配置校验。

## 文件整体说明

- **依赖与环境**：多数类使用 `@pytest.mark.usefixtures("single_rank_process_group")`，在单 rank 进程组 / CPU mesh 上验证行为；Ulysses 的 `TestAllToAll` 额外使用 `mock_all_to_all_identity_for_gloo`，将 `torch.distributed.all_to_all` 打桩为逐张量拷贝，模拟多机 all-to-all 在单测中的恒等行为。
- **DSA 侧**：覆盖自定义 autograd `ToLocalWithPartialGrad`、`allgather_sequence`、`patch_dsa_for_context_parallel` 对 `DSV32_SDPA` 的类级修改，以及 `dsa_forward_with_cp` 对 KV head 形状的约束。
- **Ulysses 侧**：覆盖 `all_to_all` / `AllToAll` 数值与 autograd、`patch_ulysses_for_context_parallel` 对 `ScaledDotProductAttentionWrapper` 的替换及对 `all_to_all` 的调用次数，以及 `validate_ulysses_configs` 的合法与非法配置。
