  全量 attention 矩阵（Full Attention）

           kv 位置 
           0  1  2  3  4  5  6  7
  query 0 [✓  ✗  ✗  ✗  ✗  ✗  ✗  ✗]                                                                                                                                                                                  
  query 1 [✓  ✓   ✗  ✗  ✗  ✗  ✗  ✗]            
  query 2 [✓  ✓   ✓  ✗  ✗  ✗  ✗  ✗]                                                                                                                                                                                  
  query 3 [✓  ✓   ✓  ✓  ✗  ✗  ✗  ✗]            
  query 4 [✓  ✓   ✓  ✓  ✓  ✗  ✗  ✗]                                                                                                                                                                                  
  query 5 [✓  ✓   ✓  ✓  ✓  ✓  ✗  ✗]                                                                                                                                                                                  
  query 6 [✓  ✓   ✓  ✓  ✓  ✓  ✓  ✗]

  每个 query 能看到自己之前的所有 token，计算量是 O(n²)。

  Band 模式（滑动窗口 attention，window_size=3）                                                                                                                                                                    
                                                                                                                                                                                                                    
  Band 模式限制每个 query 只能看到离自己最近的 window_size 个 token：                                                                                                                                               
                  
           kv 位置                                                                                                                                                                                                  
           0  1  2  3  4  5  6  7             
  query 0 [✓  ✗  ✗  ✗  ✗  ✗  ✗  ✗]                                                                                                                                                                                  
  query 1 [✓  ✓  ✗  ✗  ✗  ✗  ✗  ✗]
  query 2 [✓  ✓  ✓  ✗  ✗  ✗  ✗  ✗]                                                                                                                                                                                  
  query 3 [✗  ✓  ✓  ✓  ✗  ✗  ✗  ✗]  ← 位置 0 超出窗口，被丢弃
  query 4 [✗  ✗  ✓  ✓  ✓  ✗  ✗  ✗]                                                                                                                                                                                  
  query 5 [✗  ✗  ✗  ✓  ✓  ✓  ✗  ✗]                                                                                                                                                                                  
  query 6 [✗  ✗  ✗  ✗  ✓  ✓  ✓  ✗]                                                                                                                                                                                  
  query 7 [✗  ✗  ✗  ✗  ✗  ✓  ✓  ✓]                                                                                                                                                                                  
                                                                                                                                                                                                                    
  可以看到 ✓ 形成了一条沿对角线的斜带（band），这就是名字的来源。                                                                                                                                                   
                                                                                                                                                                                                                    
  规则就是：query i → kv[i-window_size+1 .. i]                                                                                                                                                                      
                                                                                                                                                                                                                    
  对 DeepSeek-V4，window_size=128，即每个 query 向左看 128 个 token。                                                                                                                                               
                                                                                                                                                                                                                    
  ---             
  Band 模式为什么在 CP rank r>0 下出错                                                                                                                                                                              
                                              
  问题就出在：band 模式是基于 query 在 kv 中的绝对位置来算的，它假设 query i 对应 kv[i]。                                                                                                                           
                                              
  但 CP rank r>0 的 kv_full 布局是：                                                                                                                                                                                
                                          
  kv_full = [ boundary_128(来自rank r-1) | local_chunk(rank r自己) ]                                                                                                                                                
  位置索引:      0, 1, ..., 127          | 128, 129, ..., 128+chunk-1                                                                                                                                               
                                                                                                                                                                                                                    
  kv_full[128] 才是 rank r 的第 0 个 token（全局位置 r*chunk）。                                                                                                                                                    
                                                                                                                                                                                                                    
  此时 query 0（全局位置 r*chunk）需要的窗口是 kv_full[1..128]，但 band 模式算出的是 kv_full[i-127..i] = kv_full[-127..0]，负数索引无意义，完全错误：                                                               
                                                                                                                                                                                                                    
  正确窗口（需要）:  kv_full[1 .. 128]   ← 含边界 token                                                                                                                                                             
  band 模式给的:    kv_full[-127 .. 0]  ← 完全错位                                                                                                                                                                  
                                              
  这就是为什么 CP rank r>0 必须退回 SDPA，用显式传入的 window_topk_idxs 来指定正确的访问范围，而不能依赖 SFA kernel 的 band 模式自动计算。      




[B, seqlen, n_heads, head_dim]这里的这些参数代表什么意思                                                                                   

● 这是 PyTorch 中张量的形状描述，每个维度代表一个含义：

  ---
  逐个解释

  B（batch_size）
  一次训练同时处理多少条样本。比如 B=4 表示同时处理 4 条句子，各自独立计算，互不影响。

  seqlen（sequence length）
  每条样本有多少个 token。比如句子 "我喜欢猫" 经过 tokenizer 可能变成 [我, 喜欢, 猫] 共 3 个 token，seqlen=3。训练时通常 padding
  到固定长度，比如 seqlen=4096。

  n_heads（number of attention heads）
  多头注意力的头数。Transformer 不是只做一次 attention，而是并行做 n_heads 次，每个头学习不同的注意力模式，最后合并结果。DeepSeek-V4 中
  n_heads=128。

  head_dim（head dimension）
  head_dim（head dimension）
  每个 attention head 的向量维度。通常 head_dim = total_dim / n_heads，比如总维度 7168，n_heads=128，则 head_dim=56。

  ---
  直观理解

  可以把 [B, seqlen, n_heads, head_dim] 想象成一个四维的箱子：

  外层（B）：       4 个独立的样本
    中层（seqlen）：  每个样本有 4096 个 token 位置
      中层（n_heads）：  每个位置有 128 个注意力头
        内层（head_dim）：  每个头是一个长度 56 的向量

  ---
  为什么要有这个形状

  Attention 计算的本质是：每个 token 的 query 去和所有 token 的 key 做点积，得到权重，再对 value 加权求和。

  有了形状，就能直接理解矩阵乘法在算什么：

  # query: [B, n_heads, seqlen, head_dim]
  # key:   [B, n_heads, seqlen, head_dim]

  attn = query @ key.transpose(-1, -2)
  # 结果:  [B, n_heads, seqlen, seqlen]
  #                      ↑query位置  ↑key位置
  # 每个 head、每个 query token 对所有 key token 的相似度分数





问题二：主 Loss 在哪里、如何计算                                                                                                                                                       
                                                            
  主 loss 不在模型内部计算，而是在训练循环（train.py）中计算，与 deepseek_v4_cp.py 无关。下面是完整调用链：                                                                              
                                                            
  1. CP 切分 — 在 forward 之前                                                                                                                                                           
                                                            
  train.py 的 post_dataloading_process 中（第 469 行）调用：                                                                                                                             
  inputs, labels, extra_kwargs = prepare_context_parallel_input(
      inputs, labels, ..., cp_mesh, ...                                                                                                                                                  
  )                                                                                                                                                                                      
                                              
  这会把全局 sequence [B, 4096] 按 CP 维度切分：                                                                                                                                         
  - rank 0 得到 tokens[0:2048]，labels[0:2048]              
  - rank 1 得到 tokens[2048:4096]，labels[2048:4096]                                                                                                                                     
                                                    
  2. Forward — 模型只返回 logits                                                                                                                                                         
                                                                                                                                                                                         
  train.py:forward_backward_step（第 532 行）：                                                                                                                                          
  pred = model_parts[0](inputs, **extra_inputs, **extra_kwargs)                                                                                                                          
  # model(tokens) → DeepSeekV4Model.forward() → logits [B, 2048, vocab_size]                                                                                                             
                                                                                                                                                                                         
  模型 forward 只负责产生 logits，不计算任何 loss（li_loss 也在 backward 才真正执行）。                                                                                                  
                                                                                                                                                                                         
  3. 主 Loss — Cross-Entropy，在训练循环中                                                                                                                                               
                                                                                                                                                                                         
  loss_sum = self.loss_fn(pred, labels)                                                                                                                                                  
  # = cross_entropy_loss(logits [B,2048,vocab_size], labels [B,2048])                                                                                                                    
  # = F.cross_entropy(..., reduction='sum')   ← 注意是 sum 不是 mean                                                                                                                     
                                                            
  loss = loss_sum / global_valid_tokens                                                                                                                                                  
  # global_valid_tokens = AllReduce 所有 DP rank 的 valid token 数                                                                                                                       
                                                                                                                                                                                         
  global_valid_tokens 如何保证梯度一致性：                                                                                                                                               
                                                                                                                                                                                         
  global_valid_tokens = dist_sum(local_valid_tokens, batch_mesh)                                                                                                                         
                      ≈ 4096  （cp=2 时两个 rank 各贡献 2048，但 batch_mesh 包含 CP 维度）                                                                                               
                                                                                                                                                                                         
  因此每个 token 上的梯度 = 1/global_valid_tokens ≈ 1/4096，与 cp=1 完全一样，不会因 CP 切分影响梯度大小。                                                                               
                                                            
  4. Backward — 一次调用同时触发主 Loss 和 Li Loss 的反向                                                                                                                                
                                                                                                                                                                                         
  loss.backward()                                                                                                                                                                        
                                                                                                                                                                                         
  这一行触发了两条完全独立的梯度链：                        
                                                                                                                                                                                         
  loss (CE) → model.output → norm → ... → TransformerBlock  
                                                ↓                                                                                                                                        
                              DSAIndexerLossAutoScaler.backward
                                ├── grad_x_out → 主注意力梯度链                                                                                                                          
                                │     （SFA kernel、AllGather kv_compress、P2P BE1/BE2/BE3）                                                                                             
                                └── scaled_grad → SparseLightningIndexerGradKLLossWrapper.backward
                                      ← 真正执行 NPU LiLoss kernel（ms_npu_sparse_lightning_indexer_grad_kl_loss）                                                                       
                                      → d_query_index / d_key_index / d_weights → indexer 权重梯度                                                                                       
                                      → li_loss 值存入 DSAIndexerLossLoggingHelper.tracker（打印用）                                                                                     
                                                                                                                                                                                         
  5. 日志里显示的 loss 值                                                                                                                                                                
                                                                                                                                                                                         
  训练循环末尾（第 610 行）：                                                                                                                                                            
  global_avg_loss = dist_sum(loss, loss_mesh)   # loss_mesh 包含 DP+CP 维度                                                                                                              
  # cp=2: rank0(loss=c/2) + rank1(loss=c/2) → global_avg_loss = c                                                                                                                        
                                      ← 真正执行 NPU LiLoss kernel（ms_npu_sparse_lightning_indexer_grad_kl_loss）
                                      → d_query_index / d_key_index / d_weights → indexer 权重梯度
                                      → li_loss 值存入 DSAIndexerLossLoggingHelper.tracker（打印用）

  5. 日志里显示的 loss 值

  训练循环末尾（第 610 行）：
  global_avg_loss = dist_sum(loss, loss_mesh)   # loss_mesh 包含 DP+CP 维度
  # cp=2: rank0(loss=c/2) + rank1(loss=c/2) → global_avg_loss = c
