在 Ubuntu 24.04 系统中，清理系统垃圾并不需要安装像 Windows 上那样的第三方清理软件（如 360 或管家）。通过系统自带的包管理工具 `apt` 和一些简单的命令，你就可以非常安全且彻底地清理系统缓存、冗余内核和无用的依赖包。

以下是针对开发者（特别是像你这样熟悉命令行的用户）最安全、最常用的几种系统清理方法：

### 1. 清理 APT 包管理器的缓存 (最常用)

每次你使用 `apt install` 或软件更新时，系统都会把下载的 `.deb` 安装包缓存在硬盘里。长时间不清理会占用大量空间。

- **清理已下载的包缓存（保留最新版本，删除过时的）：**
```bash
     sudo apt autoclean
```

- **彻底清空所有已下载的包缓存（最省空间）：**
```bash
    sudo apt clean
```

### 2. 删除无用的依赖包 (非常有效)

当你在编译、开发（比如配置 NPU 环境、安装各种 LLM 依赖）或者安装某些软件时，系统会自动安装许多相关的依赖包。当你后来删除了主软件，这些依赖包往往变成了“孤儿包”，白白占用空间。

- **自动删除不再被需要的依赖包：**
```bash
    sudo apt autoremove
```

> [!NOTE] 强烈建议定期运行此命令，它可以帮你释放大量因为折腾环境而残留的垃圾文件。

### 3. 清理系统日志 (释放系统盘空间)

Ubuntu 会记录详细的系统日志（通过 `journald`）。随着时间推移，这些日志文件可能会变得非常庞大。你可以限制日志占用的大小或时间。

- **查看当前系统日志占用了多少空间：**
```bash
    journalctl --disk-usage
```

- **清理日志（只保留最近 3 天的日志）：**
```bash
    sudo journalctl --vacuum-time=3d
```

- **或者按大小清理（只保留最大 100MB 的日志）：**
```bash
    sudo journalctl --vacuum-size=100M
```

### 4. 删除旧版本的 Snap 包

如果你系统里使用 `snap` 安装过软件（比如你系统里的某些工具包），Snap 默认会保留旧版本以便于你随时回滚，这会占用大量磁盘空间。你可以通过一个脚本清理它们。

- 创建一个清理脚本：
```bash
    nano clean_snap.sh
```

- 将以下内容粘贴进去：
```bash
    # !/bin/bash
    # 移除被禁用的（也就是旧版本的）snap 包
    set -eu
    LANG=en_US.UTF-8 snap list --all | awk '/disabled/{print $1, $3}' |
	    while read snapname revision; do
		    snap remove "$snapname"  --revision="$revision"
		done
```

- 赋予执行权限并运行：
```bash
    chmod +x clean_snap.sh
    sudo ./clean_snap.sh
```

---

### 💡 总结：日常维护的“一键命令”组合

为了方便，你可以把最安全的清理操作串联起来，每次觉得系统变慢或空间不足时，运行这一行就够了：
```bash
sudo apt update && sudo apt upgrade -y && sudo apt autoremove -y && sudo apt autoclean
```

这句命令会先帮你更新系统，然后自动删除无用的依赖包，最后清理安装包缓存。这也是很多高级 Linux 用户的日常保养习惯。
